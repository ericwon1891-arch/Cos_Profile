# 캐릭터 카드 그리드 "더보기" Design

**Goal:** 캐릭터 섹션(`CharacterSectionBlock`)의 카드 그리드가 한 번에 너무 많은 카드를 보여주지 않도록, 필터링된 카드가 6개를 넘으면 처음엔 6개만 보여주고 "더보기" 버튼으로 나머지를 펼칠 수 있게 한다.

**Non-goals:** 페이지네이션(여러 번에 나눠 추가 로드)은 하지 않는다 — "더보기"는 한 번 누르면 나머지 전부를 한 번에 펼친다. 별도의 재사용 컴포넌트로 분리하지 않는다 — 이 기능은 `CharacterSectionBlock` 한 곳에서만 쓰인다.

## 동작

- `CharacterSectionBlock`에 `expanded` boolean state를 추가한다 (초기값 `false`).
- 카테고리 필터링된 배열(`filtered`)의 길이가 6 초과이고 `expanded`가 `false`이면, 그리드에는 `filtered.slice(0, 6)`만 렌더링하고 그 아래 "더보기" 버튼을 표시한다.
- "더보기" 버튼 클릭 시 `expanded`를 `true`로 바꾼다 — 이후 `filtered` 전체가 그리드에 렌더링되고 버튼은 사라진다.
- `filtered.length`가 6 이하이면 애초에 버튼이 나타나지 않는다 (자르기도 하지 않음).
- 카테고리 필터 버튼을 클릭해 `activeFilter`가 바뀔 때는 `expanded`도 함께 `false`로 리셋한다 — 필터가 바뀌면 대상 카드 목록 자체가 달라지므로, 매번 6개부터 다시 보여주는 게 자연스럽다.

## 에러 처리

별도 처리 없음 — `filtered`가 빈 배열이거나 6개 이하인 일반적인 경우는 지금과 동일하게 동작(자르기 없음, 버튼 없음).

## 섹션별 온/오프 토글

이 기능은 관리자가 섹션 단위로 켜고 끌 수 있어야 한다.

- 섹션 데이터에 `showMoreEnabled` boolean 필드를 추가한다 (선택 필드). 값이 명시적으로 `false`가 아니면(즉 `true`이거나 필드 자체가 없으면) 켜진 것으로 취급한다 — 이미 마이그레이션된 기존 섹션(사진/영상)처럼 이 필드가 아예 없는 데이터도 기본 ON으로 동작해야 하기 때문.
- `showMoreEnabled`가 `false`인 섹션은 필터링된 카드 개수와 무관하게 항상 전체를 보여주고 "더보기" 버튼도 절대 나타나지 않는다.
- 관리자 폼(`CharactersForm.jsx`)에서 각 섹션의 "섹션 제목" `TextField` 바로 옆(같은 줄)에 토글 스위치(체크박스)를 둔다. 라벨은 "카드 6개로 제한" 정도로 한다.
- 새 섹션을 추가할 때 기본값은 `showMoreEnabled: true` (6개 제한이 기본 동작).

## 테스트

`CharacterSectionBlock.test.jsx`에 다음 케이스를 추가한다:
- 필터링된 카드가 6개 이하면 "더보기" 버튼이 없고 전부 표시된다
- 필터링된 카드가 7개 이상이면 처음엔 6개만 표시되고 "더보기" 버튼이 보인다
- "더보기" 클릭 시 나머지 카드가 모두 표시되고 버튼이 사라진다
- 펼쳐진 상태에서 다른 카테고리로 필터를 바꾸면 다시 6개로 접힌 상태로 시작한다
- `showMoreEnabled: false`인 섹션은 카드가 6개를 넘어도 전부 표시되고 더보기 버튼이 없다
- `showMoreEnabled` 필드가 아예 없는 섹션(레거시 데이터)은 ON으로 취급되어 6개 제한이 적용된다
